- (decision) Design paradigm: layered (api → services → pipeline|db → domain) with pipes-and-filters at pipeline/ core. Dependency direction is one-way downward; pipeline nodes never touch db directly. Binds all modules. Prevents api/pipeline cross-coupling.
- (decision) State ownership: LangGraph state (SqliteSaver) is single source of truth for pipeline data. runs table is API projection only — mirrors status, current_stage, gate_states_json. services/ layer syncs DB by consuming LangGraph stream events after each node. Pipeline nodes never write to DB directly.
- (decision) Gate mechanism: LangGraph interrupt() — each stage node calls interrupt() at completion; API resumes via Command(resume='approved'/'rejected'). gate_states dict in PipelineState is read-only projection for API queries; LangGraph checkpointer owns the actual pause/resume state.
- (decision) OQ-7 RESOLVED: PipelineState is TypedDict. ShotData uses sentence_indices: list[int] (N:M mapping, CoAgent pattern) replacing Go sentence_start/sentence_end range + cut_num. shot_id is stable shot identifier. camera_angle/camera_movement added. scenario_node prompts DeepSeek V4 as LLM-Director to decide shot boundaries by narrative/camera shift, not fixed sentence count.
- (decision) PipelineState full shape confirmed: {run_id, scp_text, scenes: list[SceneState], video_path, current_stage, gate_states: dict[str,str], prompt_variant, error}. SceneState: {scene_num, narration, shots: list[ShotData], audio_path, audio_duration, word_timings, subtitle_path}.
- (decision) OQ-3 RESOLVED: SQLModel as ORM (Pydantic+SQLAlchemy, no new dep). Single SQLite file shared with LangGraph SqliteSaver (separate tables). No scenes table — artifact paths live in LangGraph state only; local deployment makes state loss risk acceptable.
- (decision) runs table schema: id, scp_id, status, current_stage, gate_states(JSON), prompt_variant, ab_pair_id, error, started_at, updated_at. No scenes/artifacts table.
- (decision) A/B testing: two independent runs linked by ab_pair_id. POST /runs/{id}/ab creates run_B with same scp_text, prompt_variant=B, ab_pair_id=run_A.id. Evaluation reads both LangGraph states after both complete. No graph branching.
- (decision) SSE transport: asyncio.Queue per run_id (stdlib, no broker). services/ layer subscribes to LangGraph astream(), updates DB mirror + pushes to queue simultaneously. Event types: stage_entry, stage_exit, gate_pending.
- (decision) GET /scps: reads local SCP facts file (same pattern as Go scpDataPath). Serves SCP list for UI picker. Added to API surface (was missing from PRD).
- (decision) PATCH /runs/{id}/stages/{stage}/artifact: inline text edit endpoint for scenario and subtitle stages. Calls graph.update_state() to persist edit into LangGraph checkpoint + rewrites artifact file on disk. Added to API surface (was missing from PRD).
