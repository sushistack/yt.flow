- (decision) UI system: shadcn/ui + Tailwind CSS
- (decision) Theme: system-preference (light + dark both supported)
- (decision) Primary flow: SCP input → run → per-stage review (edit or retry) → approve → final video check
- (decision) A/B comparison view: occasional use, not primary
- (decision) SCP input source: /mnt/data/raw — directory of 2000 SCP items, each with facts.json (scp_id, title, object_class, rating, tags) + main.txt. UI needs a searchable picker, not a text input.
- (decision) Stage editing: text artifacts (scenario, subtitle) = inline editable; image/audio/video = retry only
- (decision) Stage navigation: sidebar (stage list) + main panel (artifact preview). Sidebar+panel layout.
- (decision) Inline text edit: save replaces artifact only; stage still requires explicit Approve to advance pipeline
- (decision) Gate notification: none — dashboard 'awaiting approval' badge is sufficient. No Web Notifications.
- (decision) Visual direction: C — Zinc System. iOS-native zinc dark (#1C1C1E ground, #2C2C2E card), system blue (#0A84FF) as sole action accent. Status: amber/purple/green/red (semantic tier, separate from accent). Card list (not table). border-radius: 12px groups, 10px CTA, 6px badges.
- (event) Key screens rendered: dashboard (run list + SCP picker modal), run-detail (image stage awaiting approval, 6-scene gallery, gate controls)
- (decision) SCP Picker search fields: (1) SCP ID numeric part ('096' → SCP-096), (2) English nickname from tags (hyphen-normalized, e.g. 'shy guy' matches tag 'shy-guy'; 'plague doctor' matches 'plague-doctor'), (3) full ID string ('SCP-096'). Default sort: rating descending. English name displayed in picker row = first non-meta tag (exclude: _licensebox, scp, _cc, featured, illustrated, rewrite, co-authored and other admin tags).
- (decision) OQ-8 resolved: all 5 stages (scenario, image, tts, subtitle, video) have gate controls. Stage count is small enough that per-stage approval is not burdensome.
- (event) spines finalized. Reviewer gate skipped. OQ-8 resolved (all 5 stages gated). Mockups promoted to mockups/. status: final on both DESIGN.md and EXPERIENCE.md.
